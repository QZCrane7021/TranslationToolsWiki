# 通用工作流

翻译视频的工具可能不尽相同，但大致流程基本一致：

1. 视频下载
2. 字幕识别
3. 字幕打轴
4. 字幕翻译

## 视频下载

视频下载这一步，许多人可能会使用网页端的 Youtube 下载工具。但是，这些工具不能录入真实的 Youtube 用户数据，或者说，合法的 cookies，结果导致下载的视频清晰度至多为 360P 或 480P。而有些运行在本地的软件要求登录 Youtube，从而能下载 Youtube 的视频，但它们通常都是基于 yt-dlp 工具套壳而来，有些软件甚至凭借用户对 yt-dlp 的不熟悉，收取会员费，借此牟利。因此，不建议使用任何不明确自己是 yt-dlp 套壳的 GUI 软件。如果你有能力，不妨直接掌握 yt-dlp 的用法。

## 字幕识别

理论上，你确实可以完全手打字幕，对于英文听力水平较好的人完全可行，但速度太慢了。因此，可以考虑使用 VideoCaptioner + FasterWhisper 来进行语音识别。这是目前效果最好的方案，但需要设备具备带有 CUDA 的 Nvidia 显卡。其他语音识别功能的提供商，要么无法输出字幕文件，要么收费高昂，要么效果很差，几乎不用考虑。

## 字幕打轴

在这一步中，译者需要前面字幕的基础上，调整起始、终止时间戳，切分长句、重新分配子句长度，以提升视频观感，并为后续字幕翻译做铺垫。字幕打轴工具有很多，有集成识别、打轴但需要会员费的 ArcTime，也有功能完备但操作复杂、代码稀碎的 Aegisub。仅对于打轴，Aegisub 完全能胜任，甚至有其他软件无法替代的优势。
