# 附录：翻译腔

以下内容部分摘自[中文维基百科](https://zh.wikipedia.org/wiki/Wikipedia:%E7%BF%BB%E8%AF%91%E8%85%94)，加上了一些自己的理解。如果你无法访问原址，则可以视本文为原百科的代理内容。但如果你完整阅读并实践了本百科的内容，就应该能访问中文维基百科了。

所谓“翻译腔”，是指带有明显外语特点的语言风格。如果译文生硬、不协调，一般就可以说有翻译腔。避免翻译腔，简而言之就是“说人话”，但“人话”和“翻译腔”究竟有什么不同呢？

!!! info "“翻译腔”不等于语言错误"
    “翻译腔”是表达上的瑕疵，不见得是错误，但还是尽量避免为好。

## 基本原则

中文较之于其他语言，有着自己的表达习惯，如句子短、子句镶嵌少、没有时态变化等。“人话”通常满足：

- 整句最多共 5 个逗号、句号，标点符号间不多于 25 字；
- 动词优先；
- 句子恰有一个主语。

当然，翻译的首要原则是符合原文的意图。如果原文中，某人说话的意图就是要用冗长的句子压倒另一个人，那作为译者，就不该自作主张地改成短句组。

## 用词习惯

### 专有名词保持一致

不仅中文，任何语言都最好能够通篇保持用词一致。

假设约翰·史密斯是丹尼尔的父亲。

!!! bug "不当的范例"
    一天，丹尼尔回到家，看到父亲在看电视。约翰坐在沙发上，翘着二郎腿，还抓着一包薯片。丹尼尔对史密斯发起了牢骚，认为……

丹尼尔的父亲是约翰，但并没有说明，导致了后文的歧义。

!!! success "正确的范例"
    一天，丹尼尔回到家，看到父亲在看电视。父亲坐在沙发上，翘着二郎腿，还抓着一包薯片。丹尼尔对父亲／他发起了牢骚，认为……

如果确实要切换用词，建议通过说明加深读者印象，也不应过多切换。

!!! success "正确的变换范例"
    一天，丹尼尔回到家，看到父亲约翰·史密斯在看电视。父亲坐在沙发上，翘着二郎腿，还抓着一包薯片。“约翰！”外面好像有人在大喊着，但是父亲全然不理会。（后文再度提起）丹尼尔整了整皱巴巴的衣领，又假装随手抄起旁边的报纸看了起来。良久，他对父亲约翰说，自己的女朋友不见了……

### 非专有名词非必要不重复

如果不是排比句，就不要重复非专有名词。

!!! bug "不当的范例"
    中文有些别的语言表达不出来的意思，别的语言也有中文表达不出来的意思。
    
这句话实际的意义是 “不同语种互换可能会导致部分表达丢失”。但无论是严格重复非专有名词，还是把总结当译文，都不合适。

!!! success "正确的范例"
    中文有些别的语言缺少的词汇，别的语言也有中文表达不出来的意思。

### 动词优先于形容词，形容词优先于名词

中文习惯用动词取代形容词，不习惯用动名词。具体来说，就是：

- 那些意思是 “令人xx的” 的形容词，在翻译成中文时往往会改为动词并接宾语，例如 “某物是令人感兴趣的” 改为 “某物激起了某具体人的兴趣”；
- “某件事的发生，令某人做出了何种反应” 在翻译成中文时往往会改为 “某件事发生了，某人做出了何种反应”。

!!! bug "不当的示例"
    - 女朋友的失踪让丹尼尔感到焦急。他额头上的汗珠引起了父亲的注意。（物作主语：失踪、汗珠）
    - 这一篇影评在当地有着极高的知名度。
    - 这部电影是令人潸然泪下的。

!!! success "正确的示例"
    - 女朋友失踪了，丹尼尔很着急，（甚至）父亲也注意到丹尼尔满头大汗。（人作主语：女朋友、父亲）
    - 这篇影评在当地非常知名。
    - 这部电影令人潸然泪下。

### 名动词直接作为谓语使用

在中文中，有些词汇既可以作为名词使用，也可以作为动词使用，比如 “表演”（同时表示进行一场表演和表演本身）。此时，名动词往往会直接作为谓语使用，而不是 “进行xx”。

!!! bug "不当的示例"
    - 现场观众可以通过短信进行投票。
    - 这么做对两人间的关系有潜在的损害。

!!! success "正确的示例"
    - 现场观众可以通过短信投票。
    - 这么做可能会损害两人间的关系。

### 能用短语概括，就尽量用短语概括

通常，除非是引用或者刻意为之，否则长语句应该尽量替换为等价的短语。

!!! bug "不当的示例"
    - 父亲站了起来，抬起胳膊，手掌猛地向丹尼尔的脸送过去，发出了“啪”的一声。丹尼尔的脸上多了道手印，他觉得很疼。
    - 警方怀疑某人涉嫌一场肇事后不顾而去的致命车祸。

这里需要注意，如果原文是刻意写长的，则应当保留。这里我们假设并非刻意。

!!! success "正确的示例"
    - 父亲站起来就是一巴掌，打得丹尼尔的脸生疼。
    - 警方怀疑某人涉嫌一场肇事逃逸的车祸命案。

但同时，**不应该因此而引入地方方言或者大众不熟悉的互联网迷因**。

!!! bug "矫枉过正的示例"
    - 父亲站起来就是一个大逼斗，扇得丹尼尔的脸贼疼。
    - 警方怀疑某人涉嫌向李启铭同学致敬。

### 省略在外文中需要解释，但中文中不言自明的内容

外文，尤其是英文，词源广泛而复杂，导致实际意义相关的语素可能在字面上大相径庭。例如，pitch、bitumen 和 asphalt 都是沥青，纵使其实际意义有细微差异，但多数情况下，这种差异已经包含在上下文中，或者根本不重要。有时候，外文原句可能会对此加以解释，而翻译成中文时，这种解释往往会省略。

!!! bug "不当的示例"
    丹尼尔喜欢在燕麦粥里加一把葡萄干，即干燥的葡萄。

葡萄干和葡萄在英文中是 raisin 和 grape，但在中文中不言自明。

!!! success "正确的示例"
    丹尼尔喜欢在燕麦粥里加一把葡萄干。

### 修改形容词的顺序

在中文中，多重形容词的排列顺序习惯和外文不同。通常，中文采用这样的形容词顺序：

**数量　　主观形容词　　国家　　颜色　　形状　　材料　名词**

一个　　奇奇怪怪的　　德国　　蓝色　　圆形　　木　　桌子

!!! bug "不当的范例"
    门口处站着来自甲国的三个害羞女生。

!!! success "正确的范例"
    门口处站着三个害羞的甲国女生。

此处省略了无用介词“来自”。下一小节会提及这一改动。

### 紧密排列形容词

在中文中，形容词嵌入句子时，要省略部分内容，而不是直接填写词典上的含义项。比如表国籍的形容词，“来自某国”的来自就往往省略。再比如形容词普遍存在的“的”，也会结合语境舍去。

!!! bug "不当的范例"
    谷仓旁边有栋小的白色的屋子。

!!! success "正确的范例"
    谷仓旁边有栋白色小屋。

### 不堆叠过长修饰语

修饰要适当后置，否则单句可能过长。

!!! bug "不当的范例"
    蓝翔是1984年成立于山东济南的传授挖掘机技术的职业技术学校。

!!! success "正确的范例"
    蓝翔是传授挖掘机技术的职业技术学校，1984年成立于山东济南。

中文习惯定语前置，但过长的定语依然是不妥的。

!!! bug "极端的不当范例"
    不要使用长得让人看了一口气喘不上来的简直能够让即使做长难句分析的题目都能得到很好的分数的读者对“的”字产生语义饱和的定语。

!!! success "正确的范例"
    不要使用过长的定语，这种定语看着让人吃力。而读者即使擅做长难句分析题目，也会为此困惑，对“的”字产生语义饱和。

### 尽量直白

有些表达，虽然没有错误，但放在中文语境中很 “别扭”。译者很可能不能具体界定症结，只能凭语感修改。

!!! bug "不当的范例"
    他实在难以想象究竟是什么造成了如今的恶果。

!!! success "正确的范例"
    - 他不明白什么造成了如今的恶果。
    - 他想不通事情怎样落得今天这地步。

### 与时间有关的时态词基本不保留

中文动词大都从上下文推断，诸如“已经”、“曾经”等时态方面的词没有必要出现，除非有意让读者容易知道或强调动词对应的时间。

!!! bug "不当的范例"
    蔡元培曾于1916年担任北京大学校长，并已于1927年辞去北京大学校长的职务。

!!! success "正确的范例"
    蔡元培于1916年担任北京大学校长，1927年辞去北京大学校长的职务。

### 强调种类时一般不加数量词

英文可数名词和 a／an 绑定，这是语法要求，中文则需视情况而定。在强调种类而非具体名目时，一般不加数量词。

!!! bug "不当的范例"
    桌上有很多东西：一台电风扇，一个文具盒，还有一盏台灯。

!!! success "正确的范例"
    桌上有很多东西：电风扇、文具盒、台灯。

### 慎用并列连词

并列连词 “和” 本来只用来衔接名词性成分，应该避免连接动词、形容词、小句等。建议视情况用 “并”“且”“或” 或者其它方式变通处理。

!!! bug "不当的范例"
    - 许多企业家也帮助和支持这些慈善机构。
    - 雅典是最强大的和最发达的城邦。
    - 如果市场是完备的和充分竞争的，市场交换的结果一定是帕累托最优的。
    - 这种语言帮助程序员为小型的和大型的项目编写逻辑清晰的代码。

!!! success "正确的范例"
    - 许多企业家也帮助并支持这些慈善机构。
    - 许多企业家也帮助、支持这些慈善机构。
    - 雅典是最强大、最发达的城邦。
    - 如果市场是完备的且充分竞争的，市场交换的结果一定是帕累托最优的。
    - 这种语言帮助程序员为大小项目编写逻辑清晰的代码。
