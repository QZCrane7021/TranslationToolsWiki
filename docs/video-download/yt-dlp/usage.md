# 使用

## 基础用法

`yt-dlp` 必须要输入视频的 URL。

```shell
yt-dlp https://www.youtube.com/watch?v=[video_id]
```

理论上，`yt-dlp` 只需要这一条参数就可以输出视频，但实则并非如此。如果没有真实的用户 Cookies，那么 `yt-dlp` 可能无法输出任何视频内容，并提示：

```
Sign in to confirm you're not a bot. Use --cookies-from-browser or --cookies for the authentication.
```

## 格式筛选

在指令中添加参数 `--list-formats`，可以让程序列出全部可用的格式：

```shell
yt-dlp "https://www.youtube.com/watch?v=ms1YBju63ps" --list-formats
```

会返回一张表格：

```markdown
--8<-- "docs/appendix/yt-dlp-formats.txt"
```

其中各列是什么意思呢？

- `ID`，某种格式的代号。需要注意某些视频会添加自动翻译音轨或者多语种音轨，从而出现主格式之外的子 ID，例如 `301-11`。
- `EXT`，视频、音频的格式。
- `RESOLUTION`，视频的分辨率。如果是纯音频会在此处显示 `audio-only`。
- `FPS`，视频的帧率。
- `CH`，来源渠道标识，一般不重要。
- `FILESIZE`，媒体大小。这里的 MiB 相当于 Windows 的 MB，相邻单位之间是 1024 倍而非 1000 倍的关系。
- `TBR`，总码率。码率越高，一般视频或音频的细节越多。此处是视频和音频的总和。
- `PROTO`，传输协议，可能为 `mhtml`、`https` 或 `m3u8`，也不重要。
- `VCODEC`，视频编码。如果是纯音频会在此处显示 `audio-only`，如果是封面图则会显示 `images`。
- `VBR`，视频码率。
- `ACODEC`，音频编码，如果是纯视频会在此处显示 `video-only`。
- `ABR`，音频码率。
- `ASR`，音频采样率。音频也不是完全连续的，但频率远高于视频，一般为 44100 Hz 或者 48000 Hz。
- `MORE INFO`，额外信息。一般会在这里展示音轨的语种、视频的常用名分辨率（例如 1080P、720P）。

为什么要用 `--list-formats`？首先，默认无任何格式参数时，`yt-dlp` 会下载质量最高的一组 `webm` 格式的视频和音频，并将二者合并。这种格式某些剪辑软件无法导入，或者可能会出现错误。其次，许多 Youtube 视频的最高清晰度可能高达 4K 甚至 8K，完全没有必要，还会导致网络压力上升。

!!! info "　"
    但是 `--list-formats` 会阻断下载。因此在正式下载时不应使用这一参数。

那如何筛选特定格式的视频呢？在指令中添加这样一条参数：

```
-f 301
```

其中，当格式只是普通的数字时，`-f` 可以和数字并起来写，比如 `-f301`；如果想选定特定的纯视频和纯音频，二者之间需要用加号连接，比如 `-f303+251`；如果是特殊的子格式，则需要用英文双引号括起来，比如 `-f "301-17"`。

## 下载封面

加上 `--write-thumbnail` 参数，可以在下载时一并下载视频封面。

!!! info "AB 测"
    Youtube 允许博主上传两套不同的标题和封面，在视频发布前期观众可能看到其中的一套，博主可以比较二者的数据，决定最终采用哪一套。因此你可能会发现，自己下载完视频、封面，一段时间后重返，封面和标题出现了改动。但国外观众的用户画像不能直接套用在国内观众身上，最终上传到国内平台的封面和标题仍需自行取舍。
