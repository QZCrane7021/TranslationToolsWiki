# Github 简介

Github 是目前世界上最大的在线代码托管平台，许多开源软件也会在这里发布。只要你有任何深入的专门的软件使用需求，就迟早要和它打交道，所以现在不妨来了解一下。

## 网络连接

Github 是**境外平台**。一般来说，此类平台会被 “墙” 屏蔽，从而无法访问。但 Github 情况很特殊，大多数国内用户时而能连，时而不能。这是因为 Github 对于国内的信息行业企业极为重要，因此获批开放了部分限制。

总体来说，中西部地区访问的稳定性也比东部地区要差，许多东部地区用户可以在不开启任何加速工具的情况下稳定直连，甚至下载速度也很可观。有些学校的校园网也会对这些网站加速，甚至论数量只有少数的综合性论坛、媒体平台被封锁。

如果你仅限于访问网站，那么 [Watt Toolkit] 工具完全足够了；但如果你想高速下载，则需要使用 [VPN]。

## Repository

Repository，仓库，是 Github 代码库的基本单位。每个仓库都有唯一一个拥有者，拥有者名下的仓库也不能重名，由此，一个仓库即可对应唯一的 URL。

仓库的代码可以用 `git` 工具下载到本地，或者在本地编辑后推送到在线平台上同步，甚至可以创建独立分支用于测试和实验，但视频翻译者显然用不上这么复杂的功能。

### Issues

对于正在活跃维护的项目，遇到问题，与其动辄换工具或者忍受问题，不如向开发者提出意见。仓库同时有接收用户意见和建议的职能，为此你需要找到 Issues 界面。在这里你可以查找问题参与讨论，或者提出新的问题。一般建议使用英文并简单描述，如果不清楚某些术语用英文怎么说，也大可以询问 AI。

!!! tip "避免提出重复问题"
    Github Issues 中有个专门的标签 —— Duplicated，用来表示重复的问题。一般来说，开发者不会过分苛责重复的问题，但还是建议在提出问题前查找一下类似问题，看看是否已经解决了。

### Release

Release，即发布页面，对于本百科的受众是最重要的一部分。在这里你会看到最新版本和既往版本发布的 “二进制文件”，也就是对于系统而言可执行的程序。当然，有些仓库只会发布源代码，让用户自行编译，但这和我们就无关了。

对于 Windows，发布的二进制文件一般是 `.exe` 可执行文件或者 `.msi` 正式的 Windows 安装程序。其中前者可能是安装包，也可能是可立刻执行的程序。一般需要你选择对应自己系统的指令集版本，例如`x86/x32`、`x64`、`ARM` 等等；对于 macOS，发布的二进制文件一般是 `.pkg` 安装包或者 `.dmg` 磁盘映像，也分为 `Intel/x86`、`M Chip/ARM` 两大版本。其中 `.dmg` 的安装方法比较特殊，但在此不多赘述。
